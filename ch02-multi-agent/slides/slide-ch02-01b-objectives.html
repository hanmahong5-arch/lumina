Ch02 Multi-Agent · Learning Objectives

Objectives

After this chapter you will be able to:

学完本章后你将能够：

  1. Differentiate four agent types (Sync, Async, Fork, Remote) by their blocking behavior, tool access whitelist, and context inheritance strategy / 从阻塞行为、工具访问白名单和上下文继承策略三个维度区分四种 Agent 类型（同步、异步、Fork、远程）

  2. Walk through runAgent()'s 4-phase lifecycle (Init → Run → Summarize → Cleanup) and identify every resource acquired in Phase 1 that is released in Phase 4 / 逐步跟踪 runAgent() 的 4 阶段生命周期（初始化 → 运行 → 汇总 → 清理），并识别阶段 1 获取的每项资源在阶段 4 的释放

  3. Diagram the triple-isolation boundary (permission mode, tool filtering via ALL_AGENT_DISALLOWED_TOOLS/ASYNC_AGENT_ALLOWED_TOOLS, and context separation) for a custom async agent / 为自定义异步 Agent 绘制三重隔离边界图（权限模式、通过 ALL_AGENT_DISALLOWED_TOOLS/ASYNC_AGENT_ALLOWED_TOOLS 的工具过滤、上下文分离）

  4. Explain how buildForkedMessages() constructs byte-identical prefixes with FORK_PLACEHOLDER_RESULT to maximize prompt cache hit rate across parallel fork children / 解释 buildForkedMessages() 如何用 FORK_PLACEHOLDER_RESULT 构建字节一致的前缀以最大化并行 fork 子 Agent 的 prompt 缓存命中率

  5. Describe the Coordinator workflow from task decomposition through Worker creation to shutdown negotiation via SendMessage's structured protocol / 描述 Coordinator 工作流：从任务分解、Worker 创建到通过 SendMessage 结构化协议的关停协商

Prerequisites / 前置知识

Ch01 Core Engine (QueryEngine, query() loop, AsyncGenerator streaming model). Understanding of how tools are registered and executed within a single agent turn. / Ch01 核心引擎（QueryEngine、query() 循环、AsyncGenerator 流式模型）。理解单 Agent 轮次内工具的注册与执行。
