权限隔离 Permission Isolation

agentGetAppState() 覆写

  • 子agent拥有独立的 toolPermissionContext
  • 权限模式从 AgentDefinition.permissionMode 读取
  • 不继承父级已授予的session allow rules
  • 通过 rootSetAppState 共享根状态写入

权限模式类型

  • bubble - 权限提示冒泡到父终端 (fork默认)
  • plan - 只读计划模式, 允许ExitPlanMode
  • default - 标准权限检查
  • bypassPermissions - 跳过权限 (受限)

安全原则: 只能向上继承

子agent不能拥有比父级更宽松的权限。即使agent定义中指定了bypassPermissions, 如果父级不允许, 也会被降级

Permission context construction

// runAgent.ts - permission isolation

const agentGetAppState = () => ({

...appState,

toolPermissionContext: {

// Independent permission mode

mode: agentDefinition.permissionMode,

// Fresh session rules

sessionRules: buildAgentRules(),

},

});

 

// Cannot show prompts in async

const canShowPrompts =

canShowPermissionPrompts ?? !isAsync;

Key: 异步agent无法显示权限提示 (无终端交互), 因此必须依赖预设规则或自动决策
