核心引擎的语言抉择

Claude Code — TypeScript 端到端

  • 核心引擎 QueryEngine 用 TS 写，单语言贯穿
  • 内嵌引擎：必须与调用方同处一个 Node 进程（返回 AsyncGenerator，跨不了进程）
  • 分发即代码——@anthropic-ai/claude-code npm 包
  • 代价：放弃 OS 级原生沙箱能力

Codex — Rust 核心 + JS 薄壳

  • 核心引擎用 Rust 写，编译进独立 native 二进制
  • 引擎是可执行文件，前端是协议消费者——进程级隔离
  • 原生 OS 集成 + 单二进制分发 + 协议化外部 API
  • 押注"基础设施"，而非"内部工具"

为什么不同：这不是品味之争。语言选择是一连串工程目标推出的必然结果——它决定了下游所有架构特性的可能性边界。
