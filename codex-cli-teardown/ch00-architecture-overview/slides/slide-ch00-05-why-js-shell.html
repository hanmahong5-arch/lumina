那 codex-cli（TypeScript）是干什么的？

打开 codex-cli/package.json：它是一个薄壳，作用是 npm 分发。npm install -g @openai/codex 能用，就因为这个包存在。它的实际工作只有三步：

① 检测平台

识别 os / arch（darwin-arm64、linux-x64 …）

② 下载二进制

拉取匹配的预编译 Rust binary

③ 启动它

exec 进 native 引擎，自己退场

这是分发包装器，不是技术债

解决的是"让 Rust 引擎走 npm 安装工作流"这个非技术问题。Cypress、esbuild、SWC 都用同样的"npm 壳 + native binary"模式。

Claude Code 无需这一层

它本就是 Node 程序——分发即代码，没有跨语言绑定的复杂度。代价：放弃 native 沙箱能力。
