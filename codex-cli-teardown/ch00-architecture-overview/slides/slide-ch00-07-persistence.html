持久化栈：6 个独立 Crate

Codex — 工业化长期记忆

  • state　SQLite-backed 的目标/任务/记忆表
  • thread-store　持久化对话历史（本地或云端）
  • agent-graph-store　sub-agent 派生关系图
  • rollout / rollout-trace　推理调试跟踪
  • memories/read + memories/write　读写分离（读轻量、写严谨）

Claude Code — 人可读的简单模型

  • src/services/memory/ 约 5 个 TS 文件
  • 存为 ~/.claude/memory/ 下的 markdown
  • 无 SQLite、无 rollout 跟踪、无 agent graph
  • 记忆 = 开发者手写的 CLAUDE.md + 简单 KV，简单到能直接 cat 出来读

不同假设：Codex 假设 agent 长期、自主、跨会话工作，记忆必须工业化；Claude Code 假设 agent 以当前会话为主，跨会话状态由人维护。ch04 展开 goal_runtime 如何配合这些 crate。
