为什么核心用 Rust？三个具体答案

答案 1

原生跨平台沙箱

Landlock / Seatbelt / Restricted Token 都是 OS 级原语，要直接驱动 C syscall 与系统 API。

Rust 零开销 FFI 进 libc/syscall；seatbelt.rs:30 直接硬编码 sandbox-exec 路径。Node 只能写脆弱的 native addon，或放弃。

答案 2

单一二进制分发

agent + sandbox + network proxy + WebRTC 客户端全部编译进 ~30MB native binary。

无 Node 运行时依赖，无 npm install。目标受众含 CI 容器、企业受限环境——那里 native binary 近乎必需。

答案 3

可被多前端消费的引擎

app-server 把引擎暴露成 HTTP/WebSocket——VSCode、Web、移动端、三方 IDE 共享同一引擎。

serde + tokio 让"引擎是可执行文件、前端是协议消费者"的拓扑天然舒服。

关键洞察：三者合起来 = Codex 从一开始押的是"基础设施"，不是"内部工具"。Rust 是原生 OS 集成 + 可分发性 + 协议化 API 的同时最优解。
