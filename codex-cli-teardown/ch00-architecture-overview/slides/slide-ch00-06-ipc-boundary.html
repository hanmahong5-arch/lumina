穿越边界的两种消息

  • Submission { id, op }　客户端 → 引擎；Op 约 20 个 variant（UserInput / Interrupt / Shutdown …）
  • Event { id, msg }　引擎 → 客户端；EventMsg 约 40 个 variant（AgentMessage / TokenCount / ContextCompacted …）
  • 都是 serde-tagged enum；id 把一个 Submission 与它产生的多个 Event 关联

不穿越的东西（关键）

  • 工具执行的实际结果（引擎内消化，只外发 begin/end）
  • LLM API 的原始响应（在 client.rs 解析成 EventMsg）
  • 沙箱 / 文件系统 / 进程状态（引擎独占）
  • 模型 API key（引擎在 codex-login 管理）

架构含义：前端是不可信的。即使是受感染的 IDE 扩展，也只能发 Submission——信不信由引擎决定。
