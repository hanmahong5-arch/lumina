compile-time 嵌入 — seatbelt.rs:21

include_str!("seatbelt_base_policy.sbpl") 在编译期将策略文本嵌入 binary；codex 不依赖外部文件。攻击者污染磁盘上的策略文件无效——要改策略必须替换整个 binary。

(version 1)

; inspired by Chrome sandbox policy

; (deny default) — closed-by-default

(deny default)

(allow process-exec)

(allow process-fork)

(allow signal (target same-sandbox))

Closed-by-Default 姿势

SBPL 第一行 (deny default) 拒绝一切未显式允许的操作——与 Chrome 沙箱策略相同，注释直接给出 Chrome 源码链接。

PATH-injection 防御 — seatbelt.rs:30

  • 硬编码 /usr/bin/sandbox-exec（绝对路径）
  • 攻击者在 $PATH 前置恶意 sandbox-exec 时，"启动沙箱"调用不会被劫持
  • 这种"想到 PATH 污染"的细节防御是真正的 security-conscious engineering

Claude Code 对比：无 Seatbelt 调用，无策略文本，无进程隔离。子进程继承调用者的 macOS sandbox context（通常是完全不限制的开发者环境）。
