该用哪种沙箱?——气闸舱决策树

类比:沙箱 = 气闸舱(airlock)。宇航员进出舱不与真空直接相连,中间隔一个受控腔体——沙箱就是 agent 与你系统之间那个「进出都要过检」的中间腔。腔体越厚越安全,但每次进出越慢。选型 = 选腔体厚度。

你的场景是哪一个?

会执行 LLM 生成的命令,或接入不可信 MCP server?

→ Codex

OS 级沙箱,~100ms 起、unprivileged,刚好够厚

要批量自动跑成百上千个任务?

→ Codex

沙箱 + 批处理,每个子任务都在腔体内

高安全多租户 / CI,要完整内核级隔离?

→ gVisor / 容器

腔体更厚,代价是更重、更慢

个人开发机,信任自己写的 prompt?

→ Claude Code

权限弹窗够用,无需 OS 沙箱
