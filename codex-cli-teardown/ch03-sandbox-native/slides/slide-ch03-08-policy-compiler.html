SBPL 文本 → macOS

Seatbelt policy language；(deny default) + allow rules；传给 sandbox-exec

Landlock ruleset → Linux

内核 ruleset struct；含 path 访问权限 bitmap；commit 到 helper binary 子进程

SID/Privilege list → Windows

Restricted Token 剥权列表；PSID + PRIVILEGE 数组；传给 CreateRestrictedToken API

为什么值得单独存在：没有这一层，三个平台模块各自解释 PermissionProfile，实现会漂移产生语义不一致。集中编译强制"语义统一、平台特化"——与 LLVM IR 的设计哲学相同：统一中间表示，专用后端。
