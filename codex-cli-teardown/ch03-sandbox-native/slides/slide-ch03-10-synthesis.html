综合：这套复杂度值得吗？

工程成本

  • 6 个相关 crate，3 套平台实现 + 1 个 helper binary
  • 完整 MITM 代理（cert 生成、HTTP/SOCKS5 解析、policy 决策引擎）
  • 跨平台 policy 编译器
  • 持续 OS API 跟踪（Landlock 5.13+；macOS sandbox-exec 随版本变化）
  • 工具兼容性问题（沙箱失败需 fall-through 路径）
  • 文档 + 用户教育成本（操作被拒时需解释原因）

安全回报

  • prompt-injection 带偏 LLM 后，~/.ssh/id_rsa 仍不可读
  • 网络数据外泄需绕过 MITM——攻击门槛极高
  • 第三方 MCP server 被 compromise，影响被限定在沙箱边界
  • 可在企业受限环境运行，无需"agent 等于开发者本人"信任
  • 可接 untrusted code（CI 跑 untrusted PR 的代码审查）
  • 对个人开发者：permission prompt 够用；对企业/CI/不可信 MCP：沙箱是必需

最深的押注：Codex 投入的 sandbox 工程成本，押的是"未来的 AI agent 必须能在不可信环境跑、必须能接不可信工具、必须 audit 能合规"。如果押对，沙箱基础设施 5 年后会成为本地 agent 的入门门槛。如果押错——他们多写了 6 个 crate。
