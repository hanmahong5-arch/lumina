两套威胁模型 — 同一问题，不同假设

Claude Code — 隐式威胁模型

  • LLM 是良性的（RLHF 对齐，不主动产恶意命令）
  • 工具实现可信（Anthropic 内部审查）
  • MCP 服务器大体可信（不绝对）
  • 主要风险：LLM 误解意图 → 破坏性操作
  • 防御：permission prompt；用户作为最后闸门
  • 子进程直接继承用户全部权限
  • 无文件系统边界，无网络过滤

Codex — 显式威胁模型（写入源码注释）

  • LLM 可能部分对抗——prompt-injection、jailbreak、工具回流注入
  • 工具完全不可信——尤其第三方 MCP server
  • 网络环境完全敌意——任何 URL 可能是攻击载体
  • 主要风险：被 prompt injection 驱动去 exfiltrate 凭证、连接 C2
  • 防御：内核级强制隔离 + MITM 网络检查 + closed-by-default

核心分歧：Claude Code 把 LLM 当会犯错的合作者；Codex 把 LLM 当在受控环境运行的不完全可信组件。这一个假设差异，决定了愿意付出多少安全工程成本。
