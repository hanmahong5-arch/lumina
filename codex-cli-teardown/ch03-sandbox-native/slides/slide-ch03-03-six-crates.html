沙箱子系统：6 个 Crate 的分工

跨平台 Facade

codex-sandboxing

条件编译入口 + policy 路由；lib.rs 按 target_os 带入 bwrap/landlock/seatbelt 模块；链接为库

独立可执行文件

codex-linux-sandbox

Linux 专属 helper binary；fork+exec 桥梁；在子进程里 commit Landlock ruleset 再 exec 用户命令

Windows

codex-windows-sandbox

Restricted Token 实现；派生剥权 token；粗粒度保护

网络层

codex-network-proxy

全功能 MITM HTTP/SOCKS5 代理；含 mitm.rs、certs.rs、network_policy.rs

进程硬化

codex-process-hardening

umask、resource limits 等进程级约束；补足 Landlock/Seatbelt 不覆盖的面

执行策略

codex-execpolicy

命令级执行策略 DSL；哪些命令 auto-approve，哪些需审批，哪些 deny

为什么 Linux 沙箱是独立可执行文件：Landlock 的 "process commits to a ruleset" API 决定了必须先 fork → 子进程 commit ruleset → exec 命令。在主进程加 ruleset 会污染主进程。codex-linux-sandbox 是这个 fork-and-exec 的牺牲者。core/src/landlock.rs:22 是调用入口。
