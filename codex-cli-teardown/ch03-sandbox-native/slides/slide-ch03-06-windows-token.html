Restricted Token 原理

  • 从用户当前 token 派生新 token，移除部分 SID 和 Privilege
  • 子进程看似仍是用户身份，但缺少关键权限
  • 阻止：写 system32、elevate、使用 Debug Privilege
  • 阻止：安装 driver、修改 Windows Defender 设置
  • 来源：codex-rs/windows-sandbox-rs/ crate

Windows 的局限 — 平台诚实

  • Windows 无 Landlock 对等物——不能精细限定文件系统子树
  • 不能阻止读 ~/Documents/passwords.txt
  • 无 namespace 隔离（无 Bubblewrap 对等物）
  • seccomp-bpf 不可用
  • Codex 文档明确承认："Windows sandbox is weaker than Linux"

诚实工程：不假装跨平台一致。每个 OS 给的原语不同，Linux 能做到的在 Windows 上没有对等实现就如实说。这种承认比强行声称"完全相同保护"更可信，用户也能做出知情的平台选择。
