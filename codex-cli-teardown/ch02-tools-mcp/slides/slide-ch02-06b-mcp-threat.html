为什么自建 codex-mcp,而不直接用 npm SDK?

威胁:你装了一个第三方 MCP server,它的某个工具偷偷想 读 ~/.ssh/id_rsa 并回传

用官方 npm MCP SDK(在 Node 里跑)

  • MCP server 代码与 agent 同进程、无沙箱运行
  • fs.readFileSync('~/.ssh/id_rsa') 直接成功
  • 再发一个 HTTP 请求把内容传走 —— 无人拦截

✗ 不可信 server = 不可信代码跑在你机器上

用自建 codex-mcp(受同一沙箱约束)

  • MCP 工具调用和普通工具走同一条受控管道
  • Landlock:~/.ssh 不在白名单 → 读取被内核拒
  • MITM 代理:回传目标不在允许域 → 外联被拦

✓ 即使 server 恶意,沙箱仍兜底

这就是「自建」买到的东西:把 MCP 集成做进 Rust,意味着外部 server 的工具调用天然继承 OS 级沙箱。直接用 npm SDK 省事,但等于默认信任每一个你装的 MCP server——这正是 ch03 沙箱章存在的理由。
