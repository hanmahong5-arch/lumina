为什么不能 Vec<Box<dyn ToolHandler>>

RPITIT 使 trait 不 object-safe。无法做动态 vtable dispatch——必须编译期已知 handler 集合，通过 enum / match / monomorphize 分派。

AnyToolResult — 统一结果类型

registry.rs:107 定义统一结果包装，把各 handler 的强类型 Output 转成可统一处理的形态，再构造 EventMsg 推给前端。

Dynamic handler — 双轨制

dynamic.rs 支持用户运行时定义工具，但仍实现同一 ToolHandler trait。运行时灵活性 + 编译期类型安全并存。

架构反思：编译期已知 handler 集合 + 运行时 Dynamic 双轨制是 Rust trait 系统的合理妥协。牺牲了"配置文件加新工具"的灵活性，换来零开销 dispatch + 编译期类型安全。
