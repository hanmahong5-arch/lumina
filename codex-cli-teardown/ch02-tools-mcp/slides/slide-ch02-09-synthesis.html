综合：ToolHandler Trait 编码了六个工程决策

决策 1

Send + Sync 强制约束

工具默认运行于多线程并发上下文——为 ch04 的 64-worker batch jobs 和 v2 multi-agent 服务。

决策 2

Defensive is_mutating 默认

默认怀疑工具变更环境，approval 是默认开启姿态——呼应 ch03 的"不信任 LLM/工具"威胁模型。

决策 3

pre/post hook 是一等公民

telemetry/audit 内置进 trait——Codex 假设部署在需要审计追溯的企业/合规环境。

决策 4

create_diff_consumer streaming hook

实时 UI 反馈是 trait 设计目标——LLM 还没写完 tool call，TUI 已经显示进度。

决策 5

关联类型 type Output

每个工具有强类型化输出，不靠 serde_json::Value 裸传——Rust 类型系统充分利用。

决策 6

Native RPITIT，无 #[async_trait]

性能洁癖——每次 tool call 不付堆分配代价。Rust 1.75+ 的现代姿势。

总结：registry.rs:44–92 这 49 行是 Codex 整体哲学（显式、并发、untrusted、可审计、高性能）的浓缩表达。读懂这 49 行，就读懂了 Codex 团队的全套工程价值观。
