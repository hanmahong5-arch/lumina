Native RPITIT — 为什么不用 #[async_trait]

AGENTS.md:22 — "Discourage both #[async_trait] and #[allow(async_fn_in_trait)].

Prefer native RPITIT trait methods with explicit Send bounds on the returned future."

#[async_trait] 宏（旧方式）

  • 把 async 方法 desugar 成 Box<dyn Future>
  • 每次调用产生堆分配（heap alloc）
  • trait object-safe：可以 Vec<Box<dyn T>>
  • Rust 1.75 之前的必要妥协

RPITIT — Rust 1.75+ 原生方式

  • -> impl Future<Output=T> + Send
  • 零堆分配——caller 用具体类型，编译期 monomorphize
  • 代价：trait 不 object-safe，不能 Box<dyn ToolHandler>
  • Codex 接受此 trade-off：选性能 > object-safety

教学价值：这是 2026 年 Rust 异步 trait 的现代写法。#[async_trait] everywhere 的代码是 pre-1.75 妥协，新项目避免。dispatch 不走 vtable，所以 Slide 8 要另解决分派机制。
