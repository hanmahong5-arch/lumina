McpConnectionManager 职责（5 项）

  • spawn / 重连每个 MCP server 子进程
  • 维护各 server 暴露的 tool schema 列表
  • 路由 tool call 到正确的 server
  • 处理 server 端 elicitation（inbound "需要用户输入 OAuth code"）
  • 配置 hot reload（改 config 无需重启 codex）

模型 tool call → MCP RequestCallTool RPC 的转换：core/src/mcp_tool_call.rs（属于 core crate，非 codex-mcp）

codex-mcp/src/ 模块

  • lib.rs — 模块组装、公开 API
  • connection_manager.rs — 连接生命周期（McpConnectionManager）
  • tools.rs — MCP tool schema 转换
  • elicitation.rs — server 端 elicitation 流程

AGENTS.md:32 guidance: MCP 相关状态变更走集中入口（connection_manager）

架构含义：codex-mcp 是独立 crate，深度集成 sandbox——spawn MCP server 子进程时自动套用 codex-linux-sandbox policy。npm 现成 MCP SDK 做不到这一点，所以 Codex 自研。
