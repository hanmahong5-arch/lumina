工具系统对照：class 继承 vs Rust trait

Claude Code — TypeScript abstract class

  • 抽象机制： abstract class Tool<Input, Output> + extends
  • 方法多态： 虚函数表，运行时分派
  • 输入类型： 泛型参数 <Input> + Zod schema 验证
  • 输出类型： 泛型参数 <Output>
  • 并发安全： 程序员自律（无强制），Promise.all 可能竞态
  • Tool 描述： abstract description: string 字段
  • 可选 hook： isMutating?(input): boolean 可选方法
  • 工具数量： ~30+ 原子工具（小而多）
  • 注册： extends + register 到工具数组

Codex — Rust trait ToolHandler

  • 抽象机制： trait ToolHandler: Send + Sync + impl
  • 方法多态： 编译期 monomorphization（RPITIT）
  • 输入类型： Args struct + #[derive(JsonSchema)]
  • 输出类型： 关联类型 type Output: ToolOutput + 'static
  • 并发安全： Send + Sync 编译期强制，持有 Rc<T> 不通过编译
  • Tool 描述： 无独立字段——从 JsonSchema derive 生成（schema 兼做描述）
  • 可选 hook： Option<...> 返回值 + 默认 impl
  • 工具数量： 18 handler（大而合，功能打包）
  • 注册： impl + 编译期已知集合

最深差别：Claude Code 的并发安全靠程序员自律。Codex 的 Send + Sync 由编译器强制——工具意外持有不可 share 的状态，在 build 时就报错。
