Goal 概念 — 跨 Session 的持久意图

Codex — CreateGoalRequest

// goals.rs:1 (doc comment)

/// A persistent goal that survives session boundaries.

/// The agent will autonomously continue toward this goal

/// even across multiple sessions if token_budget permits.

// goals.rs:38

struct CreateGoalRequest {

objective: String, // 目标描述

token_budget: Option<i64>, // None = 无限制

}

  • objective 持久化到 SQLite，跨 session 存活
  • token_budget = 自主性的安全阀
  • Goal 完成 / budget 耗尽 / 用户显式取消 → 结束

goals.rs:1, goals.rs:38

Claude Code — Session-bound 模型

  • 每次 claude 命令 = 一个独立 session
  • session 结束 → 所有状态消失（除显式写入文件外）
  • 无"目标"概念；任务由用户 prompt 驱动
  • 跨 session 持久化靠用户手动维护 ~/.claude/memory/
  • agent 不会主动"继续上次没完成的工作"

本质差异：Claude Code 的意图生命周期 = 人类 turn 的生命周期。Codex goal 的生命周期 = 用户定义的目标达成，与 session 无关。
