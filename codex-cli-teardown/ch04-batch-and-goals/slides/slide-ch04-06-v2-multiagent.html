v2 Multi-Agent — 持久进程间消息传递

v2 新增的 7 个 Agent 工具 multi_agents_v2/

spawn

创建命名 sub-agent 实例，返回 agent_id

wait

阻塞直到指定 agent 完成，拿最终结果

send_message

向运行中的 agent 注入消息（异步）

close_agent

优雅终止 agent，释放资源

list_agents

枚举所有活跃 agent 及其状态

followup_task

向已有 agent 追加新任务，不重新 spawn

message_tool

agent 间结构化消息（非对话历史）

两代系统的核心差异

  • v1 批处理 ≈ xargs -P 16
  • CSV 行 → 独立进程 → 无通信 → 汇聚结果
  • worker 之间完全隔离，不可互相影响
  • v2 消息模型 ≈ OS 进程模型
  • agent 有生命周期，可被 kill/wait/signal
  • send_message 类似 Unix kill -USR1
  • 协调者 agent 可指挥多个专家 sub-agent

v2 是从"批量作业"到"分布式 agent 编排"的质变——每个 agent 成为有名字的、可通信的持久实体。
