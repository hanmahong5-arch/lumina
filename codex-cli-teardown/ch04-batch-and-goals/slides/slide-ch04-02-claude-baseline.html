基线：Claude Code 的子 Agent 模型

Claude Code — 串行、阻塞、一次一个

LLM 决策 → Task(prompt) → 启动子 agent → 等完成 → 拿结果

(同步阻塞)

  • 主 agent 在子 agent 运行时完全阻塞
  • LLM 单次 turn 只能 emit 一个 tool call
  • 100 个文件 = 主 agent 循环 100 次，或用户手动并行（不可能）
  • 适合"需要隔离上下文的搜索"，不适合"批量处理 N 个相似项"

Codex 的回答

  • spawn_agents_on_csv — CSV 驱动的 map-reduce
  • 默认 16 个并发 worker，上限 64
  • 主 agent 不阻塞，fire-and-forget
  • 结果异步收集写入 output CSV

范式差异：Claude Code 是"一对一委派"，Codex 是"一对 N 批处理"——两个截然不同的 agent 协作心智模型。
