FuturesUnordered Worker Pool — 并发原语

Codex — Semaphore + FuturesUnordered 模式

// agent_jobs.rs:24-25

let sem = Arc::new(Semaphore::new(concurrency));

let mut futures = FuturesUnordered::new();

for row in csv_rows {

let permit = sem.clone().acquire_owned().await?;

futures.push(async move {

let result = run_agent(row).await;

drop(permit); // 归还 slot

result

});

}

while let Some(r) = futures.next().await { collect(r); }

agent_jobs.rs:24–25 (核心并发原语)

为什么不用 JS Promise.all

  • Promise.all 一次 spawn 全部 — 无背压，OOM 风险
  • Semaphore 限流：最多 N 个 in-flight，其余排队
  • FuturesUnordered 按完成顺序 poll，不等最慢者
  • 任何一个 worker panic 不影响其他 permit 释放
  • tokio 异步运行时 — 真正的 M:N 线程，非 event loop

实测含义：64 个 worker 并发 30 分钟 timeout — 单次 batch job 最多可消耗 32 小时 GPU/API 时间，对账单影响极大。
