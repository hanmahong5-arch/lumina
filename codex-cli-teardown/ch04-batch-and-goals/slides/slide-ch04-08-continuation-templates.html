LazyLock + include_str! 模式

// goals.rs:49-63

static CONTINUATION_PROMPT_TEMPLATE:

LazyLock<String> = LazyLock::new(|| {

include_str!("prompts/continuation.txt")

.replace("{objective}", "{{objective}}")

.to_string()

});

static BUDGET_LIMIT_PROMPT_TEMPLATE:

为什么是编译期而非运行时？

  • 合规审计：prompt 内容随二进制版本固定，不可被外部文件替换
  • 完整性：prompt 文件缺失 → 编译报错，不是运行时崩溃
  • 无外部依赖：单一可执行文件，无需部署附属 prompt 目录
  • LazyLock 按需初始化：首次使用时展开，非启动开销

对比：Claude Code 的系统 prompt 在运行时拼装（buildSystemPrompt()），可被环境变量/配置覆盖。Codex goal prompt 是二进制内嵌，不可在部署后修改。
