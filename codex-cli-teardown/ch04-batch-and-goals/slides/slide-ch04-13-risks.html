自主性的代价 — 5 个真实风险

风险

1

失控成本：64 workers × 30 min timeout + goal 自动续命 = 无上限账单，用户睡觉时可能花掉数千美元

2

可追溯性：多 agent 并行 + 跨 session 运行，事后难以重建"agent 为什么做了这个决定"

3

意外破坏：有 goal 驱动的 agent 可能在无人监督时修改关键文件、推送代码、发送邮件

4

Prompt 注入放大：自主循环中被注入一次恶意指令，后续所有 continuation turn 都可能被劫持

5

用户预期错位：用户以为 agent "在后台帮我工作"，实际上 agent 在做完全不同的事

Codex 的工程缓解措施

✓

token_budget：硬上限，超出强制停止

✓

SQLite 事件日志：每个 goal turn 可回溯

✓

30 min worker timeout：防止单 worker 卡死

✓

TurnAborted 事件：用户 Ctrl+C 可中断自主循环

✓

编译期 prompt 模板：防止 prompt 被运行时替换

Codex 的赌注："这些风险可以被工程手段控制。" Claude Code 的赌注："不给这个能力，就没有这些风险。" 两种截然不同的安全哲学。
