worked example:你按下 Ctrl+C 之后

「中断」为什么也要排队走 channel?跟着一次 Ctrl+C 走一遍就懂了。

1

你在 IDE 里按 Ctrl+C

2

前端封装成 Submission { op: Op::Interrupt },经 tx_sub 投入提交队列 —— 和普通输入同一条队列

3

submission_loop 顺序 rx_sub.recv() 取到它(handlers.rs:962)

4

匹配 Op::Interrupt => { interrupt(&sess).await; false }(handlers.rs:973)→ 中止当前 turn

5

引擎回发 EventMsg::TurnAborted,前端据此收尾

为什么不是「带外信号」:Interrupt 是排进同一信箱的一等公民 Op,不是随时打断的 OS 信号。于是它有确定的「位置感」、行为可预测、可序列化、能跨进程——代价是它得等队列轮到它。这就是 channel 模型用「显式排队」换来的确定性。
