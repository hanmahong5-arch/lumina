类比:传送带 与 邮局信箱

同样是「驱动一个长流程」,两种并发模型的差别用两个画面就能记住。

Claude Code = 传送带(AsyncGenerator)

隐式暂停 / 恢复,单线、单一取货人

生产者 ──▶ [yield] ──▶ 消费者

　　　　 ◀── 取一个,带子才动一格

  • yield = 传送带停一下,把控制权交回取货人
  • 背压天然:消费者不取,生产就停
  • 只能同进程、单消费者(语言原生协程跨不了线程/进程)

Codex = 邮局信箱(Channel pair)

显式异步消息,可跨线程 / 多取信人

发件 ──▶ [tx_sub] ✉✉✉ [rx_sub] ──▶ 收件

事件 ◀── [rx_event] ✉✉ [send_event] ◀──

  • send / recv = 投递与取信,彼此异步解耦
  • 背压需显式设计(有界 channel),但能跨线程、多消费者
  • 正因解耦,引擎才能是独立二进制、被多前端消费

一句话记住:传送带把「谁先谁后」焊死在调用栈里(简单、同进程);信箱把它摊成可序列化的消息(灵活、可跨界)。这正是上一页那张对照表背后的物理直觉。
