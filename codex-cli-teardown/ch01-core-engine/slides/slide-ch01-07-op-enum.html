Op enum — 用户能让引擎做的全部事（protocol/src/protocol.rs）

核心对话 Op（最常用）

  • Op::UserInput { items } — 普通对话。追加 items 到当前 thread，触发推理
  • Op::UserTurn { items, model, effort, cwd, approval_policy, sandbox_policy, … } — 完整 turn 配置版，SDK 集成常用
  • Op::Interrupt — 中断当前 turn。走同一 dispatch 通道，有"位置感"（顺序保证）
  • Op::Shutdown — 退出引擎，submission_loop 退出 while

配置与状态变更 Op

  • Op::OverridePermissionProfile — 热切换权限策略（无需重启引擎）
  • Op::OverrideTurnContext — 替换当前 turn 上下文
  • Op::CompactNow — 立即触发 context compaction
  • Op::EditPreviousMessage — 编辑历史消息并重新触发推理

Realtime / 工具 / 其他 Op

  • Op::RealtimeConversationStart / Stop / Sdp — WebRTC 语音会话控制
  • Op::ToolApprovalDecision — 用户审批工具调用（allow / deny）
  • Op::Subscribe / ListMcpTools / CleanBackgroundTerminals

为什么 Op::Interrupt 走通道？

设计成 Op 而不是 tx_sub.cancel() 是关键——中断信号有"位置感"：它排在它之后发送的 Op 之前被处理，不会乱序。

Erlang 邮箱模型

所有用户操作建模为 Op enum = Erlang 邮箱模型现代化版本。优点：协议可序列化，app-server 直接走 JSON over WebSocket 暴露给 IDE 客户端。

vs Claude Code：CC 的"操作"是方法调用：submitMessage、abortController.abort。Codex 的操作是数据——所有 Op 可序列化，跨进程/语言边界传输。
