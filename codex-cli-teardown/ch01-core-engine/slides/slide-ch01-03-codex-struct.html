Codex struct — 引擎入口的 5 个字段

// core/src/session/mod.rs:369

/// queue pair: send submissions,

/// receive events.

pub struct Codex {

pub(crate) tx_sub: Sender<Submission>,

pub(crate) rx_event: Receiver<Event>,

pub(crate) agent_status:

watch::Receiver<AgentStatus>,

pub(crate) session: Arc<Session>,

pub(crate) session_loop_termination:

SessionLoopTermination,

}

tx_sub: Sender<Submission>

入站通道。调用方通过此句柄发送 Op（UserInput / Interrupt / Shutdown …）

rx_event: Receiver<Event>

出站通道。引擎产生的所有事件——约 40 种 EventMsg 变体——从此处流出

agent_status: watch::Receiver

watch channel：只关心当前状态，旧值被覆盖。TUI 渲染"running / idle / waiting"指示灯

session: Arc<Session>

共享内部状态。Arc 让多个异步任务（subscribe / submit / goal_runner）同时持有引用

session_loop_termination

Shared future：任何持有 Codex 的代码都可 .await 等待 session loop 退出（优雅关闭）

Claude Code 对照

调用方持有 QueryEngine 实例，调 submitMessage(prompt) 返回 AsyncGenerator<SDKMessage>。调用接口是方法；Codex 的接口是通道句柄。
