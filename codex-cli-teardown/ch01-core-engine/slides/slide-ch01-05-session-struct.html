Session struct — 17 个字段，4 组职责（core/src/session/session.rs:11）

Group 1 — 通道端点（4）

  • tx_event — 发事件的生产端（rx_event 对应面）
  • agent_status: watch::Sender — 状态广播发布端
  • out_of_band_elicitation_paused — MCP elicitation 时让主循环让出控制权
  • conversation_id: ThreadId — 对话唯一标识

Group 2 — 锁保护状态（5）

  • state: Mutex<SessionState> — 模型选择、权限策略、cwd
  • active_turn: Mutex<Option<ActiveTurn>> — 当前 turn 句柄；注释："at most 1 running task at a time"
  • managed_network_proxy_refresh_lock: Semaphore — 代理重建串行化
  • pending_mcp_server_refresh_config: Mutex<...>
  • mailbox_rx: Mutex<MailboxReceiver> — 保证单消费者

Group 3 — 邮箱与缓冲（3）

  • mailbox: Mailbox — 工具执行异步结果回收队列
  • idle_pending_input — 引擎 idle 时暂存用户输入
  • next_internal_sub_id: AtomicU64 — 多任务安全自增 id；原子整数暴露并发预期

Group 4 — 子系统句柄（5）

  • conversation: Arc<RealtimeConversationManager> — WebRTC 语音（Claude Code 无）
  • goal_runtime: GoalRuntimeState — 跨 turn 持久目标 + 自动 continue（Claude Code 无）
  • guardian_review_session — LLM 审批判断器
  • features: ManagedFeatures — session 生命周期内不变的 feature flag
  • services: SessionServices — exec / mcp / auth / env 依赖注入 bag

关键洞察：active_turn 注释"at most 1 running task at a time"——Codex 用 Mutex 付出显式锁代价，换来和 Claude Code JS 单线程完全相同的不变量，但同时保留了 goal_runtime / mailbox / realtime audio 真正并行跑的能力。
