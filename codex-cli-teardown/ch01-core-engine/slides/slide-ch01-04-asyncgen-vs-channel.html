AsyncGenerator vs Channel Pair — 同一问题，两套并发哲学

AsyncGenerator（Claude Code）

  • 借助 JS 语言原生协程：yield 暂停，next() 推进
  • 调用方 .return() 取消，触发 try/finally
  • 引擎是被动的——消费者不 next 就不前进 = 天然背压
  • 单消费者、单线程；不支持后台并发任务
  • 承认"JS 只有一根线程"，用语言给的机制

Channel Pair（Codex）

  • 生产与消费完全解耦：引擎自主节奏跑 submission_loop
  • 引擎推 Event 到 tx_event，消费者何时消费是消费者的事
  • 取消 = 显式 Op::Interrupt 信号走同一通道
  • 多消费者天然（broadcast）、后台任务天然（Tokio 任务）
  • 为 goal_runtime / mailbox / realtime audio 并发打开了门
