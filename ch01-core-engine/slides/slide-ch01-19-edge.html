413 连续失败

1,279 sessions / 50+ consecutive failures (up to 3,272)

reactive_compact 单次尝试后放弃 (hasAttemptedReactiveCompact = true)。极端长对话中, 压缩后的摘要本身可能仍然超限。withhold 机制确保最终错误才暴露给用户。

2

autocompact 熔断

MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3

连续3次 autocompact 失败后停止尝试, 防止浪费 API 调用。每天约 ~250K API calls 的节省。熔断后回退到 snip/micro 两级压缩。

3

fallback model 切换

FallbackTriggeredError → 清空 assistant 消息 → 插入 TombstoneMessage → 切换到 fallbackModel。保证消息交替规则 (user/assistant) 不被破坏。
