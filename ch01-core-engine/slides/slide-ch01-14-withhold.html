Withhold-then-Recover: 无感恢复

1

检测可恢复错误

413 Prompt Too Long / max_output_tokens

2

设置 withheld = true

扣留错误消息, 不 yield 给上层 (SDK/REPL)

3

尝试恢复策略

collapse_drain → reactive_compact → truncate

4

结果分流

成功: 上层永远不知道发生了错误
失败: 释放扣留的错误, 终止会话

为什么需要 Withhold?

SDK 调用方 (desktop/cowork) 收到 error 字段会立即终止会话。过早暴露可恢复错误导致不必要的 UX 中断。

覆盖的错误类型

isWithheldPromptTooLong()
isWithheldMaxOutputTokens()

两个守卫函数镜像设计, 逻辑对称
