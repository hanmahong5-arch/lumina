Ch01 Core Engine · Learning Objectives

Objectives

After this chapter you will be able to:

学完本章后你将能够：

  1. Trace the 7-step queryLoop() pipeline from message-queue processing through API call, tool execution, stop hooks, to terminal/continue decision / 追踪 queryLoop() 的 7 步管线：从消息队列处理、API 调用、工具执行、stop hooks，到终止/继续决策

  2. Enumerate QueryEngine's 8 private fields and explain the Session-vs-Turn dual-layer state model that prevents state leakage / 列举 QueryEngine 的 8 个私有字段，并解释防止状态泄漏的 Session 层 vs Turn 层双层状态模型

  3. Classify all 7 Continue reasons and 9 Terminal conditions, and predict which path the engine takes for a given error scenario / 分类全部 7 种 Continue 原因和 9 种 Terminal 条件，并对给定错误场景预判引擎的执行路径

  4. Explain the withhold-recover pattern for max_output_tokens errors, including why messages are withheld from SDK callers and the 3-attempt recovery limit / 解释 max_output_tokens 错误的 withhold-recover 模式，包括为何对 SDK 调用方扣留消息以及 3 次恢复上限

  5. Justify the AsyncGenerator choice over EventEmitter/callback chains by demonstrating its type-safe yield, natural backpressure, and cancellation semantics / 通过展示类型安全的 yield、天然背压和取消语义，论证选择 AsyncGenerator 而非 EventEmitter/回调链的合理性

Ch00 overview map. Working knowledge of TypeScript async/await, AsyncGenerator syntax, and basic familiarity with LLM API calling conventions. / Ch00 概览图。掌握 TypeScript async/await、AsyncGenerator 语法，基本了解 LLM API 调用约定。
