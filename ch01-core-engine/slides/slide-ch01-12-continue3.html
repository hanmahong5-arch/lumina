恢复路径 6-7: Hook 与预算

6

stop_hook_blocking hooks

Stop hooks 返回 blockingErrors 时触发

  • 将 blockingErrors 注入为 user message
  • 设置 stopHookActive = true
  • 模型看到错误后修正行为
  • 若 hook 返回 'stopped' → 终止 (hook_stopped)
  • 若 hook 返回 'prevented' → 终止 (stop_hook_prevented)

handleStopHooks() →
  { blockingErrors } → continue
  { stopped } → terminal

7

token_budget_continuation budget

feature('TOKEN_BUDGET') 启用时的预算续写

  • checkTokenBudget() 评估剩余预算
  • 返回 decision.nudgeMessage 引导模型
  • 注入 nudge 作为 user message
  • incrementBudgetContinuationCount()
  • 允许模型在预算内继续自主执行

budgetTracker.check() →
  { continue: true, nudge } → inject
  { continue: false } → completed
