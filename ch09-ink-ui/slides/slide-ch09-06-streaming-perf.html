Ref-based Update Pattern

// High-frequency: use ref, skip render

responseLengthRef.current += delta.length;

 

// Low-frequency: use state, trigger render

setMessages(prev => [...prev, msg]);

  • text_delta: ~100 events/sec, ref only

  • tool_use: ~1-5 events/turn, state update

  • message_stop: 1 event/turn, final render

Ink 16ms Batch Rendering

  • Ink coalesces multiple setState calls within 16ms

  • React reconciler diffs virtual terminal tree

  • Only changed ANSI escape sequences written to stdout

  • Result: smooth streaming even at high token rate

Why Not Just setState?

At 100 text_delta/s, each setState would trigger Ink reconciliation. 100 reconciliations/s would cause visible jank and CPU spike in terminal.
