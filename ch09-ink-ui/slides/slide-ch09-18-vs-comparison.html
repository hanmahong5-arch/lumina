对比: 终端UI框架

Readline (Node)

  • Built-in, zero deps

  • Line-by-line input

  • No component model

  • Manual ANSI escape

  • No streaming UI

Ink (React)

  • React component model

  • Declarative UI

  • Flexbox layout engine

  • memo / hooks ecosystem

  • Streaming-native

Blessed (Node)

  • Widget-based (DOM-like)

  • Full TUI framework

  • Mouse support

  • Heavy dependency

  • Unmaintained

Bubbletea (Go)

  • Elm architecture

  • Msg/Update pattern

  • Pure Go, fast

  • No React ecosystem

  • Different language

Design Philosophy: Why Ink Over Alternatives

Blessed 是"终端版 GUI"——面向表单和菜单；Bubbletea 是"Go 的 Elm"——优雅但语言隔离。Claude Code 选择 Ink 不是因为它最轻量，而是因为 AI agent 需要 流式增量更新 + 复杂状态组合 + 团队 React 技能复用。这是 React 的核心优势场景。

// Ink for AI Streaming UI
useAsyncIterable(stream) → 渐进式渲染, <Box flexDirection="column"> → Flexbox 布局
useMemo(expensiveRender, [deps]) → 避免重绘, render(<App/>, {patchCanvas: true})
