API 返回 413 (Payload Too Large) 时的紧急截断策略

payload too large

▶

▶

Drop from Head

从头部删除

▶

Retry

max 3 retries

// withRetry.ts - PTL recovery

function truncateHeadForPTLRetry(

messages, tokenGap

) {

// Group messages by API round

const rounds = groupByRound(messages);

// Drop rounds from head

let freed = 0;

while (freed < tokenGap) {

const round = rounds.shift();

freed += roundTokens(round);

}

return flattenRounds(rounds);

}

为什么会触发 413?

token 估算不精确(估算 vs 实际偏差)

系统提示突然增大(新 skill/plugin)

auto compact 还未来得及触发

按 Round 分组删除

API round = user msg + assistant msg + tool results。按 round 整组删除保证消息完整性(不会出现 assistant 回复但 user 问题已删的情况)。

MAX_PTL_RETRIES = 3

超过 3 次截断重试仍失败 → 触发 auto compact 兜底。
