5 个关键收获 Key Takeaways

Token 管理与上下文压缩的核心设计思想

1

4 级级联实现成本最优的渐进降级

Micro → Snip → Collapse → Auto，前 3 级零 API 成本，仅最后一级需要 LLM 调用。70%+ 的压缩需求在 L1-L3 解决。

2

info vs confirm 是压缩的基础判据

COMPACTABLE_TOOLS 白名单区分"可重现查询结果"和"不可重现确认信息"。只压缩前者，保护后者。

3

9 段模板比自由摘要更可靠

结构化 prompt 强制 LLM 覆盖所有关键维度。analysis 草稿区提升思考质量但不浪费 token。

4

缓存命中率是核心架构指标

cache_edits 服务端删除保持前缀缓存；fork 消息字节相同确保 60%+ 命中率。成本 = 可用性。

5

压缩后清理是隐藏的复杂度热点

12+ 模块状态重置 + 子代理隔离。遗漏任何一个模块都可能导致幽灵引用或状态不一致。
