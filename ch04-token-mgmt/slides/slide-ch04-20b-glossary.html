CHAPTER 04 · Key Terms

Glossary

Context Window / 上下文窗口

Max tokens per request (Claude: 200K) — 像工作台面积，越大能摊开的文件越多

Context Pressure / 上下文压力

Used-to-available token ratio; near 100% must compress — 像硬盘使用率，快满了需要清理

Token Budget / 令牌预算

Allocated allowance per operation (e.g. 50K rebuild) — 像每月生活费预算，各有额度

Snip / 裁剪

Delete oldest messages to free tokens; cheapest but lossy — 像删除旧聊天记录，空间释放信息没了

Micro Compact / 微压缩

In-place reduction of oversized tool results — 像新闻摘要，把长文章压缩成几个要点

Compaction / 压缩

Summarize conversation to reduce tokens, keep essentials — 像会议纪要，2小时讨论压成1页

Context Collapse / 上下文坍缩

Non-destructive compression; originals kept, view collapsed — 像数据库View，原始数据不变

Post-Compact Rebuild / 压缩后重建

Re-read up to 5 key files (50K budget) after compaction — 像搬家后重新布置书桌摆常用资料

Prompt Cache / 提示缓存

Reuse cached prefix across requests for ~50% cost cut — 像预制菜，常用前菜提前备好更便宜

Lost in the Middle / 中间遗忘

LLMs recall start/end better than middle of long context — 像读厚书，开头结尾记得清中间容易忘
