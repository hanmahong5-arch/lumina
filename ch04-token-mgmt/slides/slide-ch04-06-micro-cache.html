Micro Path 2: Cached 缓存感知微压缩

利用 cache_edits 服务端删除，保持 prefix 缓存命中率

消息序列 (prefix cached)

CACHED PREFIX | NEW TAIL

cache_edits 删除 cached prefix 中的旧工具结果

SHORTER PREFIX | NEW TAIL

cache_edits 机制

服务端支持对已缓存的 prefix 进行局部删除。删除旧工具结果后，前缀仍然命中缓存。

关键约束：只能删除，不能修改或插入。

// Cached micro-compact

// Only delete from cached prefix

// to preserve cache hit rate

const edits = oldResults.map(r => ({

type: 'delete',

index: r.messageIndex,

contentIndex: r.contentIndex

}));

// Send with cache_edits

apiCall({

messages,

cache_edits: edits

});

缓存命中率保障

与 time-based 路径不同，cached 路径直接通知服务端删除，不修改本地消息数组前缀部分。

prefix 结构不变 = 缓存继续命中

与 Path 1 的互补

Path 1(time-based): cold cache, 本地裁剪

Path 2(cached): hot cache, 服务端删除
