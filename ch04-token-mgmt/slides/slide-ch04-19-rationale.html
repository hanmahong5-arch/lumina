设计哲学 Design Rationale

三个核心设计决策背后的思考

渐进式降级 Graceful Degradation

4 级级联不是复杂度的堆砌，而是成本-收益的精确控制。L1-L3 零 API 成本覆盖绝大多数场景，L4(LLM 摘要)仅在必要时触发。类比网络 QoS：在带宽不足时先降低非关键流量的优先级，而非直接断开连接。

原则：能裁剪就不截断，能截断就不折叠，能折叠就不摘要

投影视图读写分离 Projection View R/W Separation

collapse store 与 REPL 消息数组分离，实现"API 看到压缩视图，UI 看到完整历史"。这是 CQRS 模式在 token 管理中的应用：写入路径(用户输入)和读取路径(API 请求)使用不同的数据视图。

原则：压缩是 API 的视图优化，不是数据的永久丢失

9 段模板约束 Template Constraint

让 LLM 自由总结会导致不可预测的信息遗漏。9 段模板是 checklist 思维的工程化：强制 LLM 回答 9 个预定义问题，确保错误记录、用户反馈、下一步计划等关键信息不被遗漏。analysis 草稿区允许自由思考但不保留到最终输出。

原则：结构化输出比自由发挥更可靠
