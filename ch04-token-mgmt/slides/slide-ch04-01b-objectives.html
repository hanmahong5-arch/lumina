Ch04 Token Management · Learning Objectives

Objectives

After this chapter you will be able to:

学完本章后你将能够：

  1. Calculate the 4 context-pressure thresholds (warning, error, auto-compact, blocking) from a given context window size using AUTOCOMPACT_BUFFER_TOKENS and WARNING_THRESHOLD_BUFFER_TOKENS / 利用 AUTOCOMPACT_BUFFER_TOKENS 和 WARNING_THRESHOLD_BUFFER_TOKENS 从给定上下文窗口大小计算 4 个上下文压力阈值（警告、错误、自动压缩、阻塞）

  2. Order the 4-level compaction cascade (Micro → Snip → Auto LLM Summary → 413 PTL Recovery) by ascending cost and invasiveness, and state each level's trigger condition / 按成本和侵入性递增排列 4 级压缩级联（微压缩 → 裁剪 → 自动 LLM 摘要 → 413 PTL 恢复），并说明每级的触发条件

  3. List the 9 sections of the compaction summary template and explain why the Analysis-Summary two-phase structure improves summary quality via Chain-of-Thought / 列出压缩摘要模板的 9 个章节，并解释 Analysis-Summary 两阶段结构如何通过思维链提升摘要质量

  4. Distinguish time-based microcompact from cached microcompact, identify which tools appear in COMPACTABLE_TOOLS, and explain how cache_edits preserve prompt cache across compaction / 区分基于时间的微压缩和基于缓存的微压缩，识别 COMPACTABLE_TOOLS 中的工具，并解释 cache_edits 如何在压缩过程中保持 prompt 缓存

  5. Trace the runPostCompactCleanup() reset chain and explain why sub-agent compactions must skip main-thread state resets (isMainThreadCompact guard) / 追踪 runPostCompactCleanup() 的重置链，并解释为何子 Agent 压缩必须跳过主线程状态重置（isMainThreadCompact 守卫）
