子代理安全 Sub-Agent Compact Safety

isMainThreadCompact 隔离检查：子代理压缩不得清除主线程状态

Main Thread vs Sub-Agent Compact 隔离

Main Thread

messages[] + 12 module states

isMainThreadCompact = true

cleanup: ALL modules

Sub-Agent

forked messages[] (独立副本)

isMainThreadCompact = false

cleanup: agent-local ONLY

// postCompactCleanup.ts

function postCompactCleanup(

isMainThreadCompact: boolean

) {

if (isMainThreadCompact) {

// Reset ALL shared states

resetClassifier();

resetDenialTracking();

resetTokenCache();

resetCollapseStore();

// ... 12+ modules

} else {

// Agent-local only

resetLocalTokenCache();

}

}

风险场景

如果子代理 compact 不小心清除了主线程的 classifierState，主线程下一次权限判定会失败。

如果清除了 denialTracking，主线程的熔断保护失效。

隔离原则

子代理使用 fork 的消息副本

子代理 compact 仅清理自己的本地缓存

主线程状态由主线程 compact 独占管理

类比：进程隔离

如同操作系统的进程隔离，子进程不应影响父进程的内存空间。
