loadMemoryPrompt() / 三模式加载

Three distinct loading paths based on deployment context

Standard Mode

Default local CLI

  • Load user MEMORY.md from ~/.claude/

  • Load project MEMORY.md from .claude/

  • Concatenate with section headers

  • Apply truncation to each

~/.claude/MEMORY.md

.claude/MEMORY.md

KAIROS Mode

Cloud / headless agent

  • Skip user-level memory

  • Project memory from env vars

  • Injected via serialized payload

  • No filesystem dependency

process.env.MEMORY

TEAMMEM Mode

Team memory sync

  • Standard + team/ directory

  • team/ files synced via watcher

  • Secret guard filter applied

  • Merged after standard memory

.claude/team/MEMORY.md

All three paths converge through truncateEntrypointContent() before injection into the system prompt. Loading is synchronous and blocks the first API call.
