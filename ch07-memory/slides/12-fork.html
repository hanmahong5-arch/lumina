Fork 提取 / Forked Extraction Agent

Dedicated sub-agent runs memory extraction in isolation with shared resources

Cache Sharing

The forked agent shares the parent's prompt cache. This yields 60%+ cache hit rate because the extraction prompt includes the full conversation history that was already cached.

Turn Limit

maxTurns = 5

Hard limit prevents runaway extraction. Typically completes in 2-3 turns: analyze, write, verify.

// forkSubagent.ts

const agent = forkSubagent({

maxTurns: 5,

shareCache: true,

tools: memoryToolSet

});

Permission Sandbox

  • Read: Glob, Grep, Read (full access)

  • Write: Only to .claude/ directory

  • No Bash, no network, no MCP

  • No user interaction (headless)

Why Fork?

Isolation prevents extraction failures from corrupting the main conversation. Shared cache makes it nearly free.
