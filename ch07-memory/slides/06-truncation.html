truncateEntrypointContent() / 入口截断策略

Dual-pass truncation ensures MEMORY.md never exceeds system prompt budget

Pass 1: Line Truncation

Split content by newlines, keep first MAX_LINES=200 lines

Fast O(n) scan, preserves line boundaries

->

Pass 2: Byte Truncation

If result exceeds MAX_BYTES=25000, hard cut at byte boundary

Safety net for lines with very long content

// memdir.ts - truncateEntrypointContent()

function truncateEntrypointContent(content: string): string {

const lines = content.split('\n');

let result = lines.slice(0, MAX_LINES).join('\n');

if (Buffer.byteLength(result) > MAX_BYTES)

result = result.slice(0, MAX_BYTES);

return result;

}
