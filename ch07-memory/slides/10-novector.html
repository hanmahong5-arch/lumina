为什么不用向量数据库 / Why No Vector DB

Three decisive factors favor LLM-based selection over embedding similarity

Scale Ceiling

200

max memory files per scope

At this scale, a manifest fits in a single LLM call. Vector indexing overhead is not justified. Full manifest scan is O(200) = trivial.

Conditional Logic

LLM can apply reasoning:

  • "User asks about deploy, but context is staging not prod"

  • "Query mentions React but project uses Vue"

  • Temporal relevance ("deprecated" memories)

Cosine similarity cannot express these conditions.

Zero Dependencies

No external infrastructure:

  • No embedding model binary

  • No vector store process

  • No index rebuild on updates

  • No dimension mismatch risk

CLI stays a single npm package.

Trade-off: Each selection costs ~$0.001 in API calls vs. free local vector lookup. At typical usage (5-20 queries/session), this is <$0.02/session.
