CHAPTER 07 · Key Terms

Glossary

Memory System / 记忆系统

File-based persistent knowledge organized by type (user, feedback, project, reference). Survives across conversations.

就像笔记本——记住用户偏好和项目状态，下次对话直接用。

Semantic Retrieval / 语义检索

Using an LLM to select the most relevant memories for the current task, rather than keyword or vector matching.

就像问秘书"这个项目相关的笔记有哪些"——秘书理解语义后筛选。

Memory Types / 记忆类型

Four categories: user (preferences), feedback (corrections), project (context), reference (documentation).

就像笔记本的分类标签——偏好、反馈、项目、参考资料各归各处。

Team Memory / 团队记忆

Shared memory directory accessible by all team members, with secret filtering to prevent credential leakage.

就像团队的共享文档——大家都能读写，但密码会被自动隐去。

Secret Guard / 秘钥守卫

Filter that prevents secrets (API keys, passwords) from being written into team memory files via pattern matching.

就像银行柜台的隐私挡板——防止敏感信息被旁人看到。

Session Storage / 会话存储

Append-only JSONL log of all messages, enabling conversation reconstruction after restart.

就像自动保存的文档——随时可以从上次断开的地方继续。

Non-Derivability / 不可推导性

Principle that only non-obvious insights should be stored as memories; facts easily derivable from code should not be persisted.

就像只记重点笔记——能从课本查到的不用抄，只记老师的独到讲解。
