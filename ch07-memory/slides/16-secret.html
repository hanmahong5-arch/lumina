Secret Guard / 敏感信息过滤

teamMemSecretGuard.ts prevents secrets from leaking into team memory files

Detection Patterns

  • API keys - sk-*, AKIA*, xoxb-*

  • Bearer tokens - Authorization: Bearer ...

  • PEM certificates - BEGIN PRIVATE KEY

  • Connection strings - postgres://*:*@

  • AWS credentials - secret access key patterns

  • JWT tokens - eyJ* base64 patterns

// teamMemSecretGuard.ts

const SECRET_PATTERNS = [

/sk-[a-zA-Z0-9]{32,}/,

/Bearer\s+[a-zA-Z0-9._-]+/,

/BEGIN\s+(RSA\s+)?PRIVATE\s+KEY/,

];

Filter Pipeline

  • Runs before team memory is synced/committed

  • Line-by-line regex scan of memory content

  • Matching lines are redacted or rejected

  • Warnings logged for developer review

Why Guard at This Layer?

The extraction LLM might inadvertently include a secret it saw in conversation. The guard is the last defense before content reaches git (shared repo).
