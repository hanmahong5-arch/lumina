selectRelevantMemories() / 语义选择器

Sonnet model evaluates manifest against user query to pick relevant topic files

// LLM Selection Call

model: "claude-sonnet"

max_tokens: 256

system: "Select relevant memories"

response_format: {

type: "json_schema",

schema: {

selectedPaths: string[]

}

}

Input to LLM

  • User's current query text

  • Manifest array (path + title + category)

  • Instruction to select up to 5 paths

Output Processing

  • Parse JSON response for selectedPaths[]

  • Read full content of each selected file

  • Inject as additional context in system prompt

  • Failures silently ignored (graceful degradation)

Why JSON Schema?

Structured output guarantees parseable response. No regex extraction needed. Schema validation catches malformed LLM output before file I/O.
