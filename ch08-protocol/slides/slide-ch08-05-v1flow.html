Environment-based session with worker JWT and hybrid transport

1. Session Init

Create session from env vars, generate session ID

->

2. Worker JWT

Request worker token from auth service

->

3. Hybrid Transport

WebSocket + polling fallback

->

4. Poll Loop

Periodic message fetch on interval

// v1 initialization sequence

const session = createFromEnv();

const jwt = await getWorkerJWT(session);

const transport = new HybridTransport({

jwt, pollInterval: STATUS_POLL

});

Limitations

  • Env vars must be set before launch
  • No cross-transport message recovery
  • Poll interval wastes bandwidth on idle
  • JWT refresh is manual

v1 remains active behind feature gate for backward compatibility with existing deployments.
