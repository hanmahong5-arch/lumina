5个协议工程模式 / 5 Protocol Engineering Patterns

Reusable patterns extracted from the bridge implementation

1

Strategy Transport / 策略传输

Abstract transport interface allows swapping WebSocket, SSE, or polling without changing message handling logic. Version-gated at init time.

2

Bounded Dedup / 有界去重

Ring buffer + Set provides O(1) idempotency checking with fixed memory. FIFO eviction ensures bounded growth.

3

Epoch + Seq Recovery / 纪元序列恢复

Monotonic sequence numbers within an epoch scope enable cross-transport recovery while epoch bumps provide clean-slate semantics.

4

Cross-Process Token Coordination / 跨进程令牌协调

Filesystem-based counter file for dead token detection eliminates thundering herd without requiring IPC or shared memory.

5

Tiered Backoff / 分层退避

Different backoff configs for connection, poll, and general retries. Each tuned to the failure mode's expected recovery time.
