v1 vs v2 协议总览 / Protocol Overview

Two generations coexist with feature-gated routing

v1: Env-Based (Legacy)

  • Session creates worker with JWT

  • HybridTransport: polling + WebSocket fallback

  • Environment variables for config

  • Worker process spawned per session

  • Poll interval: configurable

session -> worker JWT -> HybridTransport -> poll

v2: Env-Less (Current)

  • Session connects to /bridge endpoint

  • SSE (Server-Sent Events) primary transport

  • CCRClient for JWT refresh cycle

  • No environment variable dependency

  • Sequence numbers for recovery

session -> /bridge -> SSE + CCRClient -> JWT
