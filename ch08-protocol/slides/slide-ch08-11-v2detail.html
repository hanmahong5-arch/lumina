Sequence numbers, epoch bumping, and heartbeat mechanisms

Seq-Num Recovery

Cross-transport message continuity

  • Each message gets monotonic seq number

  • Client tracks last-received seq

  • On reconnect: resume from last seq

  • Server replays missed messages

  • Zero message loss across transport switches

lastSeq: 142

resume_from: 143

Epoch Bumping

Clean reconnection semantics

  • Epoch = logical session version

  • Bumped on clean reconnect

  • Old epoch messages rejected

  • Prevents stale message delivery

  • Server-initiated bump on error

epoch: 3 -> 4

seq reset: 0

Heartbeat 20s

Liveness detection

  • Server sends ping every 20s

  • Client responds with pong

  • Missed pongs trigger reconnect

  • Keeps SSE connection alive

  • Proxy/LB timeout prevention

interval: 20_000ms

timeout: 60_000ms

These three mechanisms together achieve at-least-once delivery with epoch-scoped ordering, approaching the reliability of TCP over an HTTP/SSE transport layer.
