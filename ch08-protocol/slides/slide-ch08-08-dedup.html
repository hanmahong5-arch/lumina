BoundedUUIDSet / 消息去重

O(1) ring buffer deduplication with FIFO eviction

Data Structure

  • Set<string> for O(1) lookup

  • Array ring buffer for insertion order

  • Capacity: 2000 UUIDs

  • When full: evict oldest (FIFO)

// BoundedUUIDSet

class BoundedUUIDSet {

set: Set<string>;

ring: string[2000];

cursor: number = 0;

 

has(id): O(1)

add(id): O(1)

}

Why Ring Buffer?

Unbounded Set would grow indefinitely in long sessions. Ring buffer caps memory at exactly 2000 * UUID_SIZE bytes while maintaining O(1) for both insert and lookup.

Dedup Flow

1. Message arrives with UUID

2. Check set.has(uuid) - O(1)

3. If seen: discard (duplicate)

4. If new: add to set + ring[cursor++]

5. If cursor == 2000: evict ring[0], cursor = 0

lookup + insert
