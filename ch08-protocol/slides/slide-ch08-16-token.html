跨进程死Token退避 / Dead Token Backoff

Prevents all processes from hammering auth with the same expired token

The Problem

Multiple CLI processes share the same OAuth token. When token expires, all processes simultaneously hit the auth server, causing a thundering herd.

The Solution

  • Track consecutive 401 responses per token

  • After threshold = 3 failures, mark token as "dead"

  • Dead token triggers immediate refresh instead of retry

  • State shared via filesystem lock file

Cross-Process Coordination

1. Process A gets 401, increments counter (1)

2. Process B gets 401, increments counter (2)

3. Process C gets 401, counter reaches 3

4. Counter file marks token as dead

5. All processes skip to refresh flow

6. First successful refresh resets counter
