5 Takeaways

Key lessons from the resilience engineering infrastructure

1

AsyncGenerator retry makes waiting visible -- yield heartbeat messages instead of blocking. Users stay informed and can abort at any time.

2

Foreground/background retry partitioning limits amplification factor. 3 retries cap = max 3x load on an already overloaded server.

3

Model fallback with tombstone cleanup ensures the fallback model sees clean history, not the partial garbage from the failed model.

4

JSONL append-only + parentUuid chain = zero-cost crash recovery. No WAL, no transactions, no database -- just a text file.

5

The 4-stage message filter pipeline is the hidden hero -- without it, recovered sessions would crash the API with malformed message arrays.
