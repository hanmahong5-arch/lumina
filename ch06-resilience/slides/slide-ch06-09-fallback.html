模型降级 / Model Fallback Chain

3x consecutive 529 triggers FallbackTriggeredError

1

Detect: 3x Consecutive 529

Counter tracks consecutive 529 errors. After 3 failures on the same model within one conversation turn, threshold is reached.

2

Signal: FallbackTriggeredError

Special error type thrown to break out of the retry loop. Caught by the conversation orchestrator, not the retry engine.

3

Clean: Assistant Messages to Tombstone

Incomplete assistant response from failed model is replaced with a tombstone marker. Prevents the fallback model from seeing partial/corrupt output.

4

Switch: Alternate Model

Conversation re-sent to the fallback model. The model sees clean history (tombstones, not garbled partial responses).

Why Tombstone, Not Delete?

Deleting messages breaks parentUuid chain. Tombstones preserve the chain while marking content as "from a failed attempt." The fallback model knows context was lost and can restart the interrupted action.
