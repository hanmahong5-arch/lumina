中断检测 / detectTurnInterruption()

Three interruption states based on last message in chain

Last = Assistant (Complete)

State: Normal completion

  • Assistant finished its turn
  • stop_reason = "end_turn"
  • No recovery needed
  • Ready for next user input

Last = Tool Result

State: Mid-turn crash

  • Tool executed but response never sent
  • Inject "Continue" prompt
  • Model picks up from tool result
  • Most common crash recovery

Last = User

State: Interrupted before response

  • User sent message, no assistant reply
  • Use interrupted_prompt template
  • Model sees user message + hint
  • Resumes the user's original request

Detection Logic

Walk the chain to the leaf. Check leaf.role: "assistant" with end_turn = complete; "tool_result" = mid-turn crash; "user" = pre-response crash. Simple, exhaustive, no ambiguity.
