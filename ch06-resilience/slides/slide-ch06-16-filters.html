四层消息清洗 / 4-Stage Message Filtering

Sequential pipeline: each stage fixes one class of corruption

1

filterUnresolvedToolUses

Remove tool_use blocks that have no matching tool_result. Occurs when assistant requested a tool but process crashed before execution. Without cleanup, API rejects the message array.

2

filterOrphanedThinking

Remove thinking blocks that are orphaned (no subsequent content). Extended thinking generates <thinking> before the response. If interrupted, the thinking block exists without a conclusion.

3

filterWhitespace

Remove messages that contain only whitespace or empty content blocks. These can accumulate from partial streaming responses that were interrupted mid-token.

4

Sentinel Validation

Final pass ensures message array alternates user/assistant correctly, no two consecutive same-role messages, and first message is from user. Fixes any remaining structural issues.

Order Matters

Step 1 may create empty assistant messages (all tool_use blocks removed). Step 3 then cleans those. Step 4 validates the final structure. Reordering breaks the pipeline.
