OAuth 401 恢复 / Token Refresh Flow

sentToken record, keychain check, conditional refresh

1

401 Unauthorized Received

API returns 401. Record which token was sent (sentToken) to compare against current state.

2

Check Keychain

Read current token from OS keychain. If it differs from sentToken, another process already refreshed it.

3

Already Refreshed?

If keychain token != sentToken: use the new token directly. Skip refresh. This handles concurrent requests where token was already rotated.

4

Initiate Refresh

If keychain token == sentToken: token genuinely expired. Initiate OAuth refresh flow. Store new token. Retry original request.

Race Condition Prevention

The sentToken pattern prevents double-refresh. If 5 parallel requests all get 401, only the first one refreshes. The other 4 detect the keychain changed and reuse the new token. This is critical for avoiding OAuth rate limits on the token endpoint.
