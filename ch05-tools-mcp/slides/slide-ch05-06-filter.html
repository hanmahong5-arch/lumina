前置过滤 / filterToolsByDenyRules()

Prevent model from seeing tools it cannot use

function filterToolsByDenyRules(

tools, denyRules

) {

return tools.filter(t => {

const name = stripMcpPrefix(t.name);

return !denyRules.some(

r => r.matches(name)

);

});

}

Why Pre-filter?

If the model sees a tool in the schema, it may attempt to call it. Pre-filtering removes denied tools from the API request entirely.

MCP Server-Prefix Stripping

  • MCP tools arrive as mcp__server__toolName
  • Deny rules match against the bare toolName
  • Ensures consistent deny regardless of MCP server origin

Deny Rule Sources

  • Enterprise policy: Organization-wide restrictions
  • User config: Per-user tool disable list
  • Project .claude: Project-scoped deny
  • Permission mode: Plan mode hides write tools
