MCP 韧性 / Transport Resilience

Session recovery, auth refresh, asymmetric timeouts

SessionExpired

  • HTTP 404 or JSON-RPC code -32001
  • Server lost session state
  • Auto-reconnect with new session ID
  • Retry the failed tool call
  • Max 3 reconnect attempts

Detection

Two signals merged: HTTP status 404 on POST, or JSON-RPC error code -32001 in response body.

AuthError

  • OAuth token expired
  • 15min TTL for cached tokens
  • Auto-refresh via OAuth flow
  • Re-auth if refresh fails
  • User prompt only as last resort

Token Lifecycle

Token cached with 15min TTL. On 401, check if already refreshed by another call, else initiate refresh.

Timeout Strategy

  • POST: 60s timeout for tool calls
  • GET/SSE: No timeout (long-lived stream)
  • Asymmetric by design
  • SSE reconnects on disconnect
  • Heartbeat keepalive

Rationale

POST = discrete action, should complete fast. GET/SSE = event stream, must stay open indefinitely.
