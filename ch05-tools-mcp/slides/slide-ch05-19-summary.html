5 Takeaways

Key lessons from the tool orchestration engine

1

Three-layer gating (DCE + require + isEnabled) gives fine-grained control over tool availability at build, load, and runtime.

2

Cache-aware tool ordering (built-in prefix, MCP suffix) is a zero-cost optimization that significantly improves prompt cache hit rates.

3

The 6-stage runToolUse() pipeline bakes safety into execution rather than treating it as an afterthought (Zod, speculative, permission, hooks).

4

Deferred tool loading via ToolSearch trades one extra round-trip for 6K-15K tokens saved per request -- a clear win for large tool inventories.

5

MCP transport resilience (session recovery, auth refresh, asymmetric timeouts) makes external tool integration production-grade.
