Tier 1: Deny 强制拒绝层

Fail-closed 原则 -- deny 规则优先于所有 allow 规则

1a. denyRules 规则匹配

用户配置的 deny 规则最先执行，命中即返回 denied

1b. tool.checkPermissions

每个工具自定义的权限校验函数，可返回 deny + 原因

1c. safetyCheck 路径保护

.git/ .claude/ 路径在任何模式下不可绕过

1d-1g. 模式与沙箱

planMode 写拦截 / MCP 拒绝 / 工具禁用 / 边界检查

// permissions.ts - Tier 1 logic

if (matchesDenyRule(tool, input)) {

return { type: 'denied',

reason: 'matched deny rule' };

}

const toolCheck = tool.checkPermissions?.();

if (toolCheck?.denied) {

return toolCheck;

}

if (hasSafetyCheck(paths)) {

return { type: 'ask',

reason: 'safety-check' };

}

Fail-Closed 原则

任何 deny 判定一旦命中立即返回，不进入后续层级。确保安全性不依赖于后续逻辑的正确性。
