Safety-Check 豁免保护 Immunity

特定路径在任何权限模式下都不可绕过，必须用户确认

SAFETY-CHECK PATHS

免疫于 bypass / auto / acceptEdits

保护路径

原因

.git/

版本控制核心，篡改可能丢失历史

.claude/

工具配置，被修改可能绕过安全策略

settings.json

权限规则配置，自我修改等于提权

CLAUDE.md

系统提示注入风险

// filesystem.ts - safety check

function hasSafetyCheck(paths) {

return paths.some(p =>

p.includes('.git/') ||

p.includes('.claude/') ||

p.endsWith('settings.json') ||

p.endsWith('CLAUDE.md')

);

}

// This check runs in Tier 1

// BEFORE any mode-based allow

// Result: always "ask" user

设计原理

这些路径的操作等同于安全策略的自我修改。如果 AI 能自动修改 .claude/settings.json 中的 allow 规则，就等于自我提权。

Stash-Restore 模式

对 .git 操作时，自动 stash 用户未提交更改，操作完成后 restore。防止 git 操作误伤工作区。
